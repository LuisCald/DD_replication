# Moments from coefficients

A small, self-contained corner for people who have the published coefficient
files and want to compute **their own moments** — point estimates *and*
posterior bands. Everything here wraps the canonical helpers
(`code/julia/reconstruct.jl`, `code/python/reconstruct.py`); nothing is
re-implemented, so there is a single source of truth.

Julia is the primary language (matching the rest of the package); the Python
twin is kept for parity and agrees to machine precision.

## What you need

All inputs live in [`../../data/synthetic/`](../../data/synthetic/):

| File | Needed for | Ships in repo? |
|---|---|---|
| `*_coefficients_{normal,average}.csv` | point-estimate moments | ✅ yes |
| `smoothed_factors.csv` | fitting `FactorMap` (factors → coefficients) | ✅ yes |
| `smoothed_factor_draws.csv` | **posterior bands** | ⚠️ generated by `export_draws.jl` (see below) |

## 1. Point-estimate moments

```bash
julia --project=../julia/env moments_demo.jl                 # PSID, 2008-Q3
DATASET=SCF DATE=2020-Q3 julia --project=../julia/env moments_demo.jl
```

Prints consumption/income/wealth deciles and the copula density at the joint
median, and writes `moments_demo_output.csv`. (Python twin: `python moments_demo.py`.)

## 2. Posterior bands on a moment

The published draws file `smoothed_factor_draws.csv` stacks the smoothed
factors over posterior draws of the model parameters. `FactorMap` maps any
factor vector to a coefficient row, so pushing each draw through it gives a
posterior distribution over **any** moment.

```bash
julia --project=../julia/env posterior_bands.jl              # wealth deciles, 2008-Q3
MEASURE=income DATE=2020-Q3 LO=0.10 HI=0.90 julia --project=../julia/env posterior_bands.jl
```

Writes `posterior_bands_output.csv` and `posterior_bands_output.png`.
(Python twin: `python posterior_bands.py`.)

## 3. Plot the factors with posterior bands

```bash
julia --project=../julia/env plot_factor_bands.jl            # first 8 factors, 5–95%
N_FACTORS=12 LO=0.10 HI=0.90 julia --project=../julia/env plot_factor_bands.jl
```

Writes `factor_posterior_bands.{pdf,png}`. (Python twin: `python plot_factor_bands.py`.)

## Generating the posterior draws (once, on the estimation machine)

The draws file is produced from the DIME sampler output of Stage 2:

```bash
# from repo root, after Stage 2 has written posterior_draws/<measures>_<tag>.jld2
julia --project=code/julia/env code/julia/export_draws.jl
# or bundled into the post-estimation run:
julia --project=code/julia/env code/julia/run_postestimation.jl   # export step is on by default
```

This reconstructs the factors at a sample of posterior draws (one Kalman
smoother pass each) and writes `data/synthetic/smoothed_factor_draws.csv`
plus a per-quarter percentile summary `smoothed_factors_bands.csv`. Control
the count with `DD_N_DRAWS` (default 200). See
[`../julia/PosteriorDraws.jl`](../julia/PosteriorDraws.jl).

> The raw parameter draws (`posterior_draws/*.jld2`, hundreds of MB) are **not**
> shipped in git — θ is not directly usable downstream, and the compact factor
> draws above carry the same uncertainty in a form that plugs straight into
> these scripts.
